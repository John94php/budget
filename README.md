[comment]: <> (1.  **`/node_modules`**: This directory contains all of the modules of code that your project depends on &#40;npm packages&#41; are automatically installed.)

[comment]: <> (2.  **`/src`**: This directory will contain all of the code related to what you will see on the front-end of your site &#40;what you see in the browser&#41; such as your site header or a page template. `src` is a convention for “source code”.)

[comment]: <> (3.  **`.gitignore`**: This file tells git which files it should not track / not maintain a version history for.)

[comment]: <> (4.  **`.prettierrc`**: This is a configuration file for [Prettier]&#40;https://prettier.io/&#41;. Prettier is a tool to help keep the formatting of your code consistent.)

[comment]: <> (5.  **`gatsby-browser.js`**: This file is where Gatsby expects to find any usage of the [Gatsby browser APIs]&#40;https://www.gatsbyjs.com/docs/reference/config-files/gatsby-browser/&#41; &#40;if any&#41;. These allow customization/extension of default Gatsby settings affecting the browser.)

[comment]: <> (6.  **`gatsby-config.js`**: This is the main configuration file for a Gatsby site. This is where you can specify information about your site &#40;metadata&#41; like the site title and description, which Gatsby plugins you’d like to include, etc. &#40;Check out the [config docs]&#40;https://www.gatsbyjs.com/docs/reference/config-files/gatsby-config/&#41; for more detail&#41;.)

[comment]: <> (7.  **`gatsby-node.js`**: This file is where Gatsby expects to find any usage of the [Gatsby Node APIs]&#40;https://www.gatsbyjs.com/docs/reference/config-files/gatsby-node/&#41; &#40;if any&#41;. These allow customization/extension of default Gatsby settings affecting pieces of the site build process.)

[comment]: <> (8.  **`gatsby-ssr.js`**: This file is where Gatsby expects to find any usage of the [Gatsby server-side rendering APIs]&#40;https://www.gatsbyjs.com/docs/reference/config-files/gatsby-ssr/&#41; &#40;if any&#41;. These allow customization of default Gatsby settings affecting server-side rendering.)

[comment]: <> (9.  **`LICENSE`**: This Gatsby starter is licensed under the 0BSD license. This means that you can see this file as a placeholder and replace it with your own license.)

[comment]: <> (10. **`package-lock.json`** &#40;See `package.json` below, first&#41;. This is an automatically generated file based on the exact versions of your npm dependencies that were installed for your project. **&#40;You won’t change this file directly&#41;.**)

[comment]: <> (11. **`package.json`**: A manifest file for Node.js projects, which includes things like metadata &#40;the project’s name, author, etc&#41;. This manifest is how npm knows which packages to install for your project.)

[comment]: <> (12. **`README.md`**: A text file containing useful reference information about your project.)
